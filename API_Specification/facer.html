Applications

This service provides information about the applications in the system

/applications get

get

Get applications based on the Query Parameters

get /applications

Get applications based on the Query Parameters

  • Request
  • Response

Query Parameters

  • userId: (string - minLength: 3 - maxLength: 15)

HTTP status code 200

This call retrieves list of applications based on the Query Parameters.

Body

Media type: application/json

Type: array of Application

Items: Application

  • id: required (integer)

    This is the unique id of the application

  • name: required (string - minLength: 3 - maxLength: 50)

    This is the name of the application.

  • status: required (one of Green, Yellow, Red)

    This is the status of the application.

Example:

[
	{
		"id": 1,
		"name": "ARMS",
		"status": "Green"
	},
	{
		"id": 2,
		"name": "Cashiering",
		"status": "Red"
	},
	{
		"id": 3,
		"name": "HAN",
		"status": "Yellow"
	},
	{
		"id": 4,
		"name": "IRIS",
		"status": "Red"
	}
]

HTTP status code 422

This is a application error

Body

Media type: application/json

Type: object

Properties
  • code: required (number)

    This is the specific application error code

  • message: required (string)

    This is the specific application error message

  • requestParameters: (string)

    These are the request parameters recieved in the original request

Example:

{
  "code": 100,
  "message": "Invalid user id specified.",

}

Components

This resource provides access to the applciation components

/components get

get

Get list of components based on the Query Parameters

get /components

Get list of components based on the Query Parameters

  • Request
  • Response

Query Parameters

  • userId: (string - minLength: 3 - maxLength: 15)
  • applicationId: (string - minLength: 3 - maxLength: 15)

HTTP status code 200

This call retrieves list of components based on the Query Parameters.

Body

Media type: application/json

Type: array of Component

Items: Component

  • id: required (string - minLength: 1 - maxLength: 11)

    Unique identfier of the component

  • name: required (string - minLength: 1 - maxLength: 80)

    This is the short name of the application component.

  • longName: required (string - minLength: 3 - maxLength: 80)

    This is the long name of the application component.

  • type: required (string - minLength: 3 - maxLength: 20)

    This is the type of the application component. Examples are Database, Application, Reports

  • status: required (one of Green, Yellow, Red)

    This is the status of the component.

  • applicationId: required (string)

    Application Id to which the component is associated with

  • active: required (boolean)

    This tells whether the component is enabled in the system or not

Example:

[{
  "id": "1",
  "name": "Web Database",
  "longName": "This is the Web Database of Cashiering",
  "type": "Database",
  "status": "Green",
  "applicationId": "1",
  "active": true
},
{
  "id": "2",
  "name": "Payment Manager",
  "longName": "This is Payment manager application",
  "type": "Application",
  "status": "Yellow",
  "applicationId": "1",
  "active": true
},
{
  "id": "3",
  "name": "Apache F5",
  "longName": "This is the Apache F5 box",
  "type": "Network",
  "status": "Green",
  "applicationId": "1",
  "active": true
}]

Incidents

This service provides the information about the incidents in the system

/incidents get

get

Gets list of incidents based on the Query Paramters

get /incidents

Gets list of incidents based on the Query Paramters

  • Request
  • Response

Query Parameters

  • userId: (string - minLength: 3 - maxLength: 15)
  • applicationId: (string - minLength: 3 - maxLength: 15)
  • componentId: (string - minLength: 3 - maxLength: 15)

HTTP status code 200

This is the list of all incidents matching to the query criteria

Body

Media type: application/json

Type: array of Incident Object

Items: Incident Object

  • createdBy: required (string)

    User/System creating the Incident

  • componentId: required (string)

    Component to which the incident is associated with

  • applicationName: required (string)

    Aplication name to which the incident is associated with

  • lastUpdatedDate: required (datetime-only)

    Date & Time the incident was last updated

  • type: required (one of Unavailability, Slowness)

    Type of the Incident

  • typeId: required (string)

    Id of the incident type

  • componentName: required (string)

    Component name to which the incident is associated with

  • title: required (string)

    Title or short description of the incident

  • applicationId: required (string)

    Aplication to which the incident is associated with

  • resolvedDate: required (datetime-only)

    Date & Time the incident was resolved

  • details: required (string)

    Detailed description of the incident

  • id: required (string - minLength: 3 - maxLength: 15)

    Unique identifier for the Incident

  • severity: required (one of SEVERE, MEDIUM, MILD)

    Priority of the Incident

  • createdDate: required (datetime-only)

    Date & Time the incident was created

  • assignedTo: required (string)

    User to which the incident is assigned to

  • lastUpdatedBy: required (string)

    User/System last updated the Incident

Example:

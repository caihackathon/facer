Applications

This service provides information about the applications in the system

/applications get

get

Get applications based on the Query Parameters

get /applications

Get applications based on the Query Parameters

Secured by basic
  • Request
  • Response
  • Security

Query Parameters

  • userId: (string - minLength: 3 - maxLength: 15)

HTTP status code 200

This call retrieves list of applications based on the Query Parameters.

Body

Media type: application/json

Type: array of Application

Items: Application

  • name: required (string - minLength: 3 - maxLength: 50)

    This is the name of the application.

  • status: required (one of Green, Yellow, Red)

    This is the status of the application.

Example:

[{"name":"IRIS","status":"Green"},
{"name":"BACP","status":"Yellow"},
{"name":"Cashiering","status":"Red"}]

HTTP status code 401

Authentication error

HTTP status code 403

Authorization error

HTTP status code 422

This is a application error

Body

Media type: application/json

Type: object

Properties
  • code: required (number)

    This is the specific application error code

  • message: required (string)

    This is the specific application error message

  • requestParameters: (string)

    These are the request parameters recieved in the original request

Example:

{
  "code": 100,
  "message": "Invalid user Id.",
  "requestParameters": "/123/applications?status=Red"
}

Secured by basic

Headers

  • Authorization: required (string)

    The Authorization field value consists of credentials containing the authentication information of the user. Example "Authorization: Basic c2ViaXM6WipcbWokKEtcJlQnNGNZVlVkSHA="

  "status": "Yellow"

Incidents

This service provides the information about the incidents in the system

/incidents get

get

Gets list of incidents based on the Query Paramters

get /incidents

Gets list of incidents based on the Query Paramters

Secured by basic
  • Request
  • Response
  • Security

Query Parameters

  • userId: (string - minLength: 3 - maxLength: 15)
  • applicationId: (string - minLength: 3 - maxLength: 15)
  • componentId: (string - minLength: 3 - maxLength: 15)

HTTP status code 200

This is the list of all incidents matching to the query criteria

Body

Media type: application/json

Type: array of Incident Object

Items: Incident Object

  • createdBy: required (string)

    User/System creating the Incident

  • lastUpdatedDate: required (datetime-only)

    Date & Time the incident was last updated

  • type: required (one of Unavailability, Slowness)

    Type of the Incident

  • title: required (string)

    Title or short description of the incident

  • details: required (string)

    Detailed description of the incident

  • priority: required (one of High, Medium, Low)

    Priority of the Incident

  • id: required (string - minLength: 3 - maxLength: 15)

    Unique identifier for the Incident

  • createdDate: required (datetime-only)

    Date & Time the incident was created

  • assignedTo: required (string)

    User to which the incident is assigned to

  • lastUpdatedBy: required (string)

    User/System last updated the Incident

Secured by basic

Headers

  • Authorization: required (string)

    The Authorization field value consists of credentials containing the authentication information of the user. Example "Authorization: Basic c2ViaXM6WipcbWokKEtcJlQnNGNZVlVkSHA="
